Mathematical Framework

Discrete Mathematics Deep Dive

The theoretical foundations powering every calculation in the simulator.

Permutations & Combinatorics

A permutation is an ordered arrangement. With n candidates, voters can express their preferences in n! distinct orderings. The number of ways to rank r out of n candidates uses the partial permutation formula.

P(n, r) = n! / (n − r)!
// 4 candidates, all ranked: P(4,4) = 4!/0! = 24
  • Every unique vote ordering is a permutation of candidates
  • Total preference space has n! elements
  • Used to enumerate all possible election outcomes

Graph Theory

The election is modeled as a directed weighted graph G = (V, E). Each candidate is a vertex. A directed edge (A → B) with weight w means w voters prefer A over B. The graph reveals dominance cycles (Condorcet paradox).

G = (V, E, w)
// V = {A,B,C,D}, E ⊆ V×V, w: E → ℕ
  • Vertices represent candidates
  • Directed edges model preference direction
  • Edge weights = number of voters
  • Condorcet winner has all outgoing edges dominant

Inclusion-Exclusion Principle

When counting voters who prefer multiple candidates simultaneously, naive addition double-counts the overlap. The inclusion-exclusion principle corrects this for any number of overlapping sets.

|A∪B∪C| = |A|+|B|+|C| − |A∩B| − |A∩C| − |B∩C| + |A∩B∩C|
// Generalises to n sets via alternating sum
  • Prevents double-counting overlapping voter groups
  • Used in multi-winner and coalition analysis
  • Generalises via Möbius inversion

Relations & Properties

A preference relation R ⊆ C × C over candidates C captures pairwise preferences. Analysing its algebraic properties reveals voting consistency and detects paradoxes like the Condorcet cycle.

aRb ⟺ majority prefer a over b
// Transitivity: aRb ∧ bRc ⟹ aRc
  • Reflexive: every candidate preferred over itself
  • Transitive: preference chains are consistent
  • Asymmetric: if A≻B then ¬(B≻A)
  • Total: every pair is comparable

Voting System Fairness

Arrow's Impossibility Theorem (1951) proves no ranked voting system satisfies all fairness criteria simultaneously. We compare systems against: unanimity, independence of irrelevant alternatives, and non-dictatorship.

Borda(c) = Σᵢ (n − rankᵢ(c))
// Higher position = more points
  • Plurality: simple majority, susceptible to spoilers
  • Borda Count: weighted position scoring
  • IRV: eliminates last-place iteratively
  • Condorcet: winner beats all others head-to-head

Simulation Model

Voters are generated stochastically with weighted preference profiles, producing realistic ballot distributions with varying degrees of consensus and strategic voting patterns. Bias sliders let you model real-world scenarios.

weights ~ Bias(α) + Uniform noise
// α controls consensus & popularity bias
  • Each voter ranks all candidates randomly
  • Preference weights model realistic distributions
  • Seeded RNG enables reproducible analysis
  • Scenario presets for paradox hunting
