Core Concepts

Mathematical Foundations

The national grid is modelled as G = (V, E, W) — a weighted, directed graph where substations are vertices, transmission lines are edges, and impedance/capacity are weights.

📐

Weighted Graph G(V,E,W)

Each substation is a vertex V. Each transmission line is an edge E with weight W representing power loss, impedance, or distance.

G = (V, E, w: E → ℝ⁺)
🔍

Dijkstra's Algorithm

Finds the optimal (minimum-loss) transmission path between any two grid nodes. Used for load dispatch routing and emergency re-routing during faults.

d[v] = min(d[v], d[u] + w(u,v))
🗺️

Chinese Postman Problem

Finds a minimum-weight closed walk covering every transmission line at least once. Critical for inspection tours and maintenance routing of the grid.

Min Σw(e) s.t. every e traversed ≥1
🌲

Minimum Spanning Tree

Kruskal's / Prim's algorithm identifies the minimum-cost backbone of the grid — the spanning tree with lowest total transmission loss.

MST: min Σw(e), T is a tree spanning V
🌊

Max-Flow / Min-Cut

Ford-Fulkerson theorem applied to find the maximum power that can be transmitted between source (generator) and sink (city) without overloading any line.

max-flow = min-cut (Ford-Fulkerson)
📊

Degree & Centrality

Node degree identifies high-risk hubs. Betweenness centrality finds substations most critical to grid connectivity — prime fault targets.

Bc(v) = Σ σ(s,t|v) / σ(s,t)

Algorithm Details

Each algorithm addresses a distinct operational challenge in national grid management.

Dijkstra's Shortest Path

Grid Application: Given a weighted graph of substations and transmission lines, find the minimum-loss path to route power from any generator to any demand node.

In power grids, edge weights represent transmission loss (MW), impedance (Ω), or line length (km). Dijkstra's guarantees the globally optimal route under non-negative weights.

Time: O((V+E) log V) Space: O(V)
  • Initialize all distances to ∞ except source (d[s]=0)
  • Use a min-priority queue ordered by tentative distance
  • Relax edges: d[v] = min(d[v], d[u] + w(u,v))
  • Extract minimum at each step; mark visited
  • Reconstruct path via predecessor array π[v]
  • Applied to emergency re-routing when lines fail
JavaScriptdijkstra.js
// Dijkstra's algorithm for power grid routing
// graph: adjacency list {node: [[neighbor, weight],...]}
// src: source substation ID
function dijkstra(graph, src) {
  const dist = {}, prev = {}, visited = new Set();
  const pq = new MinPriorityQueue();

  // Init: all distances = Infinity
  for (const node of Object.keys(graph)) {
    dist[node] = Infinity;
    prev[node] = null;
  }
  dist[src] = 0;
  pq.enqueue(src, 0);

  while (!pq.isEmpty()) {
    const { element: u } = pq.dequeue();
    if (visited.has(u)) continue;
    visited.add(u);

    for (const [v, weight] of graph[u]) {
      if (visited.has(v)) continue;
      // Relaxation step — core of Dijkstra
      const alt = dist[u] + weight;
      if (alt < dist[v]) {
        dist[v] = alt;
        prev[v] = u;
        pq.enqueue(v, alt); // Update priority
      }
    }
  }
  return { dist, prev };
}

// Reconstruct path from prev[] array
function getPath(prev, target) {
  const path = [];
  let cur = target;
  while (cur !== null) {
    path.unshift(cur);
    cur = prev[cur];
  }
  return path; // [src, n1, n2, ..., target]
}

// Example: Route from Delhi (0) to Mumbai (7)
// Returns path + total transmission loss in MW
const { dist, prev } = dijkstra(gridGraph, "Delhi");
const path = getPath(prev, "Mumbai");
console.log(`Path: ${path.join(' → ')}`);
console.log(`Loss: ${dist["Mumbai"]} MW·km`);

Chinese Postman Problem

Grid Application: A maintenance crew must inspect every transmission line exactly once (or more) and return to the depot. The Chinese Postman finds the shortest such tour.

For an Eulerian graph (all even-degree nodes), the tour simply follows an Euler circuit. For non-Eulerian graphs, we duplicate minimum-weight edges connecting odd-degree nodes using matching.

Time: O(V³) with matching Euler circuit: O(E)
  • Find all odd-degree vertices in the graph
  • Find minimum-weight perfect matching on odd vertices
  • Add duplicate edges from matching to make graph Eulerian
  • Find an Euler circuit in the augmented graph (Hierholzer's)
  • Tour cost = original edge sum + duplicate edge sum
  • Guarantees minimum total distance for full grid inspection
JavaScriptchinese_postman.js
// Chinese Postman Algorithm
// Step 1: Find odd-degree vertices
function findOddVertices(graph) {
  return Object.entries(graph)
    .filter(([v, edges]) => edges.length % 2 !== 0)
    .map(([v]) => v);
}

// Step 2: Min-weight perfect matching on odd vertices
// (uses Dijkstra between all odd-vertex pairs)
function minWeightMatching(oddVerts, graph) {
  const pairs = [];
  const distances = {};
  // Compute all-pairs shortest paths
  for (const v of oddVerts) {
    distances[v] = dijkstra(graph, v).dist;
  }
  // Greedy matching (exact: Blossom algorithm)
  let remaining = [...oddVerts];
  while (remaining.length > 0) {
    const u = remaining[0];
    let best = Infinity, bestV = null;
    for (const v of remaining.slice(1)) {
      if (distances[u][v] < best) {
        best = distances[u][v];
        bestV = v;
      }
    }
    pairs.push([u, bestV, best]);
    remaining = remaining.filter(
      x => x !== u && x !== bestV
    );
  }
  return pairs;
}

// Step 3: Hierholzer's Euler Circuit
function eulerCircuit(graph, start) {
  const adj = JSON.parse(JSON.stringify(graph));
  const circuit = [], stack = [start];
  while (stack.length) {
    const v = stack[stack.length - 1];
    if (adj[v] && adj[v].length) {
      stack.push(adj[v].pop()[0]);
    } else {
      circuit.push(stack.pop());
    }
  }
  return circuit.reverse();
}

// Full Chinese Postman solver
function chinesePostman(graph) {
  const totalWeight = sumAllEdges(graph);
  const oddVerts = findOddVertices(graph);
  if (oddVerts.length === 0) {
    // Already Eulerian — just find circuit
    return { tour: eulerCircuit(graph, "depot"),
             cost: totalWeight };
  }
  const matching = minWeightMatching(oddVerts, graph);
  const extraCost = matching.reduce(
    (s,[,, w]) => s + w, 0
  );
  // Add matched paths to graph, then find circuit
  const augmented = addMatchedEdges(graph, matching);
  return { tour: eulerCircuit(augmented, "depot"),
           cost: totalWeight + extraCost };
}

Kruskal's MST Algorithm

Grid Application: Find the minimum-cost set of transmission lines that keeps all substations connected. Used in new grid expansion planning to minimise infrastructure cost.

Kruskal's sorts all edges by weight and greedily adds the cheapest edge that doesn't form a cycle, using a Union-Find (Disjoint Set Union) data structure.

Time: O(E log E) Space: O(V)
  • Sort all transmission lines by weight (cost/loss) ascending
  • Initialize Union-Find: each substation in own set
  • For each edge (u,v): if find(u) ≠ find(v), add to MST
  • Union(u,v) merges the two components
  • Stop when MST has V−1 edges
  • Result: backbone grid with minimum total transmission cost
JavaScriptkruskal_mst.js
// Union-Find (Disjoint Set Union) data structure
class DSU {
  constructor(n) {
    this.parent = Array.from({length:n}, (_,i) => i);
    this.rank   = new Array(n).fill(0);
  }
  find(x) {
    if (this.parent[x] !== x)
      this.parent[x] = this.fn.find(this.parent[x]);
    return this.parent[x];
  }
  union(x, y) {
    const [rx, ry] = [this.find(x), this.find(y)];
    if (rx === ry) return false; // Same component
    if (this.rank[rx] < this.rank[ry])
      this.parent[rx] = ry;
    else if (this.rank[rx] > this.rank[ry])
      this.parent[ry] = rx;
    else { this.parent[ry] = rx; this.rank[rx]++; }
    return true;
  }
}

// Kruskal's MST for national grid backbone
function kruskalMST(nodes, edges) {
  // Sort edges by weight (transmission loss / cost)
  edges.sort((a, b) => a.weight - b.weight);

  const dsu = new DSU(nodes.length);
  const mstEdges = [];
  let totalCost = 0;

  for (const { u, v, weight } of edges) {
    if (dsu.union(u, v)) { // No cycle formed
      mstEdges.push({ u, v, weight });
      totalCost += weight;
      if (mstEdges.length === nodes.length - 1)
        break; // MST complete: V-1 edges
    }
  }
  return { mstEdges, totalCost };
}

// Grid planning: find minimum backbone
// nodes = substations, edges = possible transmission lines
const { mstEdges, totalCost } = kruskalMST(
  substations, transmissionLines
);
console.log(`MST backbone: ${mstEdges.length} lines`);
console.log(`Min cost: ${totalCost} MW·km`);

Max-Flow / Min-Cut (Ford-Fulkerson)

Grid Application: Find the maximum power (MW) that can be transmitted from a generation hub to a demand centre without violating any line's capacity constraint.

The Min-Cut theorem tells us the bottleneck — the set of transmission lines whose removal disconnects generator from load. These are the critical lines to upgrade or protect.

Time: O(V · E²) BFS Min-cut = Max-flow
  • Model each transmission line as directed edge with capacity c(u,v)
  • Find augmenting path from source S to sink T using BFS (Edmonds-Karp)
  • Push flow equal to bottleneck capacity along path
  • Update residual graph: reduce forward, increase reverse
  • Repeat until no augmenting path exists
  • Min-cut identifies the critical lines for grid hardening
JavaScriptmax_flow.js
// Edmonds-Karp (BFS-based Ford-Fulkerson)
// For finding max power flow in the grid
function bfsPath(residual, src, sink, parent) {
  const visited = new Set([src]);
  const queue = [src];
  while (queue.length) {
    const u = queue.shift();
    for (const [v, cap] of residual[u]) {
      if (!visited.has(v) && cap > 0) {
        visited.add(v); parent[v] = u;
        if (v === sink) return true;
        queue.push(v);
      }
    }
  }
  return false; // No augmenting path
}

function maxFlow(graph, src, sink) {
  // Build residual graph (forward + reverse edges)
  const residual = buildResidual(graph);
  let flow = 0;
  const parent = {};

  while (bfsPath(residual, src, sink, parent)) {
    // Find bottleneck capacity along path
    let pathFlow = Infinity;
    let v = sink;
    while (v !== src) {
      const u = parent[v];
      pathFlow = Math.min(pathFlow,
        getCapacity(residual, u, v));
      v = u;
    }
    // Augment flow along path
    v = sink;
    while (v !== src) {
      const u = parent[v];
      updateResidual(residual, u, v, pathFlow);
      v = u;
    }
    flow += pathFlow;
  }
  // Find min-cut: S-side = reachable from src in residual
  const minCut = findMinCut(residual, src);
  return { maxFlow: flow, minCut };
}

Formal Grid Graph Model

The national transmission network is modelled as a weighted undirected graph G = (V, E, w) where every element has a precise engineering interpretation.

V (vertices)Substations, generating stations, load centres — each node in the network
E (edges)High-voltage transmission lines connecting pairs of substations
w(e)Edge weight: transmission loss (MW), impedance (Ω), or thermal limit (MVA)
deg(v)Number of lines connected to substation v; high degree = critical hub
Path PSequence of substations and lines from generator to load centre
Cycle CClosed loop in the grid; provides redundancy for fault tolerance
Cut setEdges whose removal disconnects the grid; critical vulnerability set
Spanning TreeMinimum lines keeping all nodes connected; backbone with no cycles
Euler circuitPath traversing every line once; optimal maintenance inspection route
Connectivity κMinimum nodes to remove to disconnect grid; resilience metric
